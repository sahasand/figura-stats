# Explore Plot — interactive ggplot2 builder (design)

Date: 2026-07-15
Status: approved

## Purpose

A third guided analysis, **Explore plot**, that turns the app into an interactive
ggplot2 workbench: the user uploads a CSV, maps columns to aesthetics
(x / y / color / facet), picks a geom, and tweaks options — and the plot
re-renders live via webR. The text pane shows the **exact, reproducible
ggplot2 code** for the current plot, making the module double as a ggplot2
starter/teaching tool. Prior art for the UX is the esquisse package
(Shiny point-and-click ggplot builder); this design reimplements that idea
on the app's existing spec → webR → SVG contract.

## Decisions made during brainstorming

| Question | Decision |
|---|---|
| Scope | Full plot builder (esquisse-style), not a fixed-chart param tweaker |
| Update model | Auto re-render, debounced (~400 ms), with latest-wins cancellation |
| Chart types (v1) | Core six: scatter, line, boxplot, violin, bar, histogram/density |
| UI shell | The guided three-stage shell (`createGuidedShell`), like KM and Summary |
| Text pane | Reproducible ggplot2 code (not a caption) |
| R design | Code-first: generate the code string, then eval that exact string |

## Architecture

Follows the five-parallel-keys recipe from CLAUDE.md:

1. **`R/explore.R`** — `fig_explore(spec)` returning `list(svg =, text =)`,
   where `text` is the reproducible ggplot2 code string.
2. **`R/dispatch.R`** — add `explore = fig_explore(spec),` to the switch.
3. **`web/worker.js`** — add `"explore.R"` to the boot-time R-file fetch list.
   **No `EXTRA_PACKAGES` entry**: all six geoms use only ggplot2, which is in
   the boot set — the explorer costs zero extra package download.
4. **`web/guided/explore/`** — shell config + content + demo files mirroring
   `web/guided/summary/`; registered in `web/app.js`; nav button in
   `web/index.html`.
5. **`DESCRIPTION`** — no change (no new packages).

Journal export (DPI-stamped PNG, SVG, copy/tsv) works with no new code via the
existing pane-header toolbars, which read `#preview`/`#stats` at click time.

**Spec shape**

```json
{
  "figure": "explore",
  "data": [ {"age": 61, "arm": "Treatment", ...}, ... ],
  "roles": { "x": "age", "y": "biomarker", "color": "arm", "facet": null },
  "options": {
    "geom": "scatter",
    "pointSize": 2, "alpha": 0.8, "smoother": "lm", "se": true,
    "title": "", "xlab": "", "ylab": "",
    "propBar": false, "position": "dodge",
    "bins": 30, "density": false,
    "notch": false, "jitter": false,
    "innerBox": true, "trim": false,
    "linewidth": 0.8, "showPoints": false
  }
}
```

Only the options relevant to the selected geom are honored; the rest are
ignored by R and hidden by the UI.

## R side — code-first rendering

`fig_explore(spec)`:

1. **Validate.** Geom must be one of the whitelisted six. Role columns are
   type-checked per geom with readable `stop()` messages, reusing the
   suppress-warning-then-clear-`stop()` coercion pattern (`.numeric_col` in
   `R/summarize.R`). Missing required roles and unknown geoms produce
   readable errors through the never-throws `render_figure` contract.
2. **Generate code.** Build the ggplot2 code string from the validated spec:
   `ggplot(df, aes(...))` + geom layer + optional `facet_wrap()` + `labs()` +
   scale/palette + theme lines.
3. **Eval that exact string** in a minimal `new.env()` containing only the
   data frame. The rendered plot cannot drift from the displayed code because
   the code is what ran. The eval input is generated by our own R function
   from a validated spec — never raw user text.
4. **Return** `list(svg = .svg_string(p, w, h), text = code)`, where the
   displayed code is standalone: `library(ggplot2)`, a
   `# df <- read.csv("your-data.csv")` header comment, then the plot code —
   copy-paste runnable in the user's own R session.

### Geoms, roles, and options (v1)

| Geom | Required roles | Options |
|---|---|---|
| scatter | x numeric, y numeric | point size, alpha, smoother (none/lm/loess) + CI band on/off |
| line | x numeric, y numeric | linewidth, points on/off |
| boxplot | x categorical, y numeric | jittered points overlay, notch |
| violin | x categorical, y numeric | inner boxplot, trim |
| bar | x categorical | count vs proportion; dodge vs stack (only when color set) |
| histogram | x numeric | bins slider; histogram vs density toggle |

Shared across all geoms:

- **color** (optional, categorical) → house Tol palette (consistent with
  `R/km.R .km_palette`; hoist/share the palette rather than redefining it).
- **facet** (optional, categorical) → `facet_wrap(vars(...))`.
- Title / x-label / y-label text inputs → `labs()`.
- House theme (from `R/themes.R`).

All generated code is ggplot2 4.x-clean: `linewidth` not `size` for lines,
no deprecated geoms/args. **WARN 0 is a hard gate.**

## JS side — guided shell + live loop

`web/guided/explore/guided-explore.js` is a `createGuidedShell({...})` config,
never a shell copy. Stage content:

- **Understand** (`content.js`): what aesthetic mapping is (columns → x / y /
  color / facet), and when to use which of the six geoms.
- **Try an Example** (`demo.js`, `demo-data.js`): frozen synthetic clinical
  dataset generated by `data-raw/explore-demo-generator.R` (~120 patients:
  age, BMI, biomarker, arm, sex). The "experiment controls" for this analysis
  are the actual builder controls.
- **Analyze Your Data** (`analyze-form.js`): CSV upload via `web/lib/csv.js`,
  role dropdowns via `web/lib/columnpicker.js` (type-filtered per geom), same
  builder controls as the demo stage.

**Live update loop** — extends the shell's experiment-controls + `rerun()`
pattern with:

- **Debounce**: ~400 ms after the last control change before sending the spec.
- **Latest-wins**: a monotonically increasing request id per module; responses
  carrying a stale id are dropped, so the single-threaded R worker never
  paints an out-of-date plot. A subtle "Rendering…" indicator shows on the
  preview pane while a request is in flight.

**Geom-driven control visibility**: switching geom swaps which role pickers
and option controls are visible (boxplot has no bins slider). Role selections
incompatible with the new geom are cleared; the column pickers' type filtering
guards most invalid specs before R sees them.

## Errors and testing

- **R** (`tests/testthat/test-explore.R`):
  - One golden-path test per geom: SVG renders (`<svg` prefix) and the code
    string contains the expected calls (e.g. `geom_boxplot`, `facet_wrap`).
  - Error tests: unknown geom, non-numeric column in a numeric role, missing
    required role → readable messages via the never-throws contract.
  - Run with `devtools::test()`; `[ FAIL 0 | WARN 0 ]` required.
- **JS unit** (`npm run test:unit`): spec-building and control-visibility
  logic (`analyze-form.test.mjs` style); frozen demo-data snapshot test.
- **E2E** (Playwright): load the explorer, demo renders, change geom + one
  option, assert the plot re-renders and the code text updates, and the
  export toolbar enables.

## Out of scope (v1)

- Heatmaps, error-bar summary plots, strip charts (candidate v2 geoms).
- Numeric color scales / continuous gradients.
- Multiple layered geoms beyond the built-in combos (e.g. violin + inner box).
- Saving/loading plot configurations.
